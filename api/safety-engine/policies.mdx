The Safety Engine provides a comprehensive set of pre-built policies for content filtering and protection. Each policy combines a rule (detection logic) and an action (what to do when detected).

## Policy Types

Policies are named using this pattern: `[Category][Action]Policy[_Variant]`

**Actions:**
- `Block` - Blocks content with a user-friendly message
- `Anonymize` - Replaces sensitive data with random values (reversible)
- `Replace` - Replaces sensitive data with placeholders like `[REDACTED]`
- `RaiseException` - Raises `DisallowedOperation` exception

**Variants:**
- `_LLM` - Uses LLM for enhanced action messages
- `_LLM_Finder` - Uses LLM for both detection and action

## PII (Personal Identifiable Information) Policies

Protects personal data like names, emails, addresses, phone numbers, and government IDs.

### Detected Patterns

- **Email addresses**: `john@example.com`
- **Phone numbers**: US and international formats
- **Social Security Numbers**: `123-45-6789`
- **Credit card numbers**: All major card formats
- **Addresses**: Street addresses and postal codes
- **Dates of birth**: Multiple date formats
- **Driver's licenses**: Various state formats
- **Passport numbers**: International formats
- **IP addresses**: IPv4 and IPv6
- **MAC addresses**: Network hardware addresses

### Available Policies

<CodeGroup>

```python Block Content
from upsonic.safety_engine import PIIBlockPolicy

agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[PIIBlockPolicy]
)

# Blocks content containing PII
result = agent.run("My email is john@example.com and SSN is 123-45-6789")
# Raises block error with user-friendly message
```

```python Anonymize PII
from upsonic.safety_engine import PIIAnonymizePolicy

agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[PIIAnonymizePolicy]
)

# Anonymizes PII before sending to LLM
result = agent.run("Send email to john@example.com")
# LLM sees: "Send email to xkqp@fhsmwtr.lzn"
# Response is de-anonymized: "Email sent to john@example.com"
```

```python Replace with Placeholder
from upsonic.safety_engine import PIIReplacePolicy

agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[PIIReplacePolicy]
)

# Replaces PII with [PII_REDACTED]
result = agent.run("My email is john@example.com")
# LLM sees: "My email is [PII_REDACTED]"
```

```python Raise Exception
from upsonic.safety_engine import PIIRaiseExceptionPolicy

try:
    agent = Direct(
        model="openai/gpt-4o",
        safety_policies=[PIIRaiseExceptionPolicy]
    )
    result = agent.run("My SSN is 123-45-6789")
except DisallowedOperation as e:
    print(f"Policy violation: {e}")
```

```python LLM-Enhanced Detection
from upsonic.safety_engine import PIIBlockPolicy_LLM_Finder

# Uses LLM for better context-aware PII detection
agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[PIIBlockPolicy_LLM_Finder]
)
```

</CodeGroup>

### PII Policy Reference

| Policy | Description | Use Case |
|--------|-------------|----------|
| `PIIBlockPolicy` | Blocks content with PII | Prevent PII from being processed |
| `PIIBlockPolicy_LLM` | Block with LLM message | Enhanced user feedback |
| `PIIBlockPolicy_LLM_Finder` | LLM detection + block | Maximum accuracy |
| `PIIAnonymizePolicy` | Reversible anonymization | Process while protecting PII |
| `PIIReplacePolicy` | Replace with placeholder | Simple redaction |
| `PIIRaiseExceptionPolicy` | Raise exception | Strict enforcement |

## Profanity Detection Policies

Detects toxic content, profanity, and inappropriate language using ML models.

### Detection Models

Powered by [Detoxify](https://github.com/unitaryai/detoxify) library with multiple models:

- **`original`**: BERT-based model from Toxic Comment Classification
- **`unbiased`**: RoBERTa-based, minimizes unintended bias (default)
- **`multilingual`**: XLM-RoBERTa for multiple languages
- **`original-small`**: Lightweight Albert-based version
- **`unbiased-small`**: Lightweight unbiased version

### Toxicity Categories

- `toxicity` - Overall toxic score
- `severe_toxicity` - Extremely toxic content
- `obscene` - Obscene language
- `threat` - Threatening language
- `insult` - Insulting content
- `identity_attack` - Attacks on identity groups
- `sexual_explicit` - Sexually explicit content

### Available Policies

<CodeGroup>

```python Default Unbiased Model
from upsonic.safety_engine import ProfanityBlockPolicy

# Uses 'unbiased' model by default
agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[ProfanityBlockPolicy]
)
```

```python Multilingual Detection

    model="openai/gpt-4o",
```


</CodeGroup>

### Profanity Policy Reference

**Block Policies:**
- `ProfanityBlockPolicy` - Default (unbiased model)
- `ProfanityBlockPolicy_Original` - Original BERT model
- `ProfanityBlockPolicy_Multilingual` - Multilingual support
- `ProfanityBlockPolicy_OriginalSmall` - Lightweight original
- `ProfanityBlockPolicy_UnbiasedSmall` - Lightweight unbiased
- `ProfanityBlockPolicy_LowThreshold` - More sensitive (0.3 threshold)
- `ProfanityBlockPolicy_HighThreshold` - Less sensitive (0.7 threshold)
- `ProfanityBlockPolicy_CPU` - Force CPU processing
- `ProfanityBlockPolicy_GPU` - Force GPU processing

**LLM-Enhanced Block Policies:**
- `ProfanityBlockPolicy_LLM` - LLM-enhanced messages
- `ProfanityBlockPolicy_LLM_Original` - Original + LLM
- `ProfanityBlockPolicy_LLM_Multilingual` - Multilingual + LLM
- (Plus all other variants with `_LLM` suffix)

**Exception Policies:**
- `ProfanityRaiseExceptionPolicy` - All variants with exception action
- `ProfanityRaiseExceptionPolicy_LLM` - LLM-enhanced exceptions

## Anonymization Patterns

The Safety Engine uses reversible anonymization to protect sensitive data:

### How Anonymization Works

1. **Detection**: Rule detects sensitive data (e.g., `john@example.com`)
2. **Anonymization**: Replaced with random values preserving format (e.g., `xkqp@fhsmwtr.lzn`)
3. **LLM Processing**: LLM sees anonymized version
4. **De-anonymization**: Response is restored to original (e.g., `john@example.com`)

### Character Replacement Rules

- **Digits** → Random digits (`123` → `847`)
- **Uppercase letters** → Random uppercase (`ABC` → `XKQ`)
- **Lowercase letters** → Random lowercase (`abc` → `pzn`)
- **Special characters** → Preserved (`@`, `-`, `.` stay the same)

### Example

```python
from upsonic.safety_engine import PIIAnonymizePolicy

agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[PIIAnonymizePolicy]
)

# Original input
input_text = "Send payment to john@example.com for account 1234-5678"

# What LLM sees (anonymized)
# "Send payment to xkqp@fhsmwtr.lzn for account 8472-3951"

# LLM response
# "Payment sent to xkqp@fhsmwtr.lzn from account 8472-3951"

# De-anonymized output
result = agent.run(input_text)
# "Payment sent to john@example.com from account 1234-5678"
```

## Combining Multiple Policies

You can apply multiple policies simultaneously:

```python
from upsonic.safety_engine import (
    PIIAnonymizePolicy,
    FinancialInfoAnonymizePolicy,
    ProfanityBlockPolicy,
    CryptoBlockPolicy
)

agent = Direct(
    model="openai/gpt-4o",
    safety_policies=[
        PIIAnonymizePolicy,           # Anonymize PII
        FinancialInfoAnonymizePolicy, # Anonymize financial data
        ProfanityBlockPolicy,         # Block profanity
        CryptoBlockPolicy             # Block crypto content
    ]
)

# PII and financial data are anonymized
# Profanity and crypto content are blocked
```

## Policy Execution Order

Policies are executed in the order specified:

1. **Input Processing**: Each policy's rule checks the input
2. **Action Execution**: If rule triggers, action is taken
3. **Blocking**: If any policy blocks, execution stops
4. **Anonymization**: Transformations are applied cumulatively
5. **LLM Call**: Modified input is sent to LLM
6. **De-anonymization**: Response is restored using transformation maps

## Exception Handling

```python
from upsonic.safety_engine import PIIRaiseExceptionPolicy, DisallowedOperation

try:
    agent = Direct(
        model="openai/gpt-4o",
        safety_policies=[PIIRaiseExceptionPolicy]
    )
    result = agent.run("My SSN is 123-45-6789")
except DisallowedOperation as e:
    # Handle policy violation
    print(f"Safety policy violation: {e}")
    # Log, alert, or take corrective action
```
