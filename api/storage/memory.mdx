## Memory

The `Memory` class serves as the central coordinator for memory operations in Upsonic, managing both session memory (chat history, summaries) and user memory (user profiles and traits).

### Key Design Principle

Session memory type is selected at **runtime**, not at initialization time. The same Memory instance can be shared across Agent, Team, and Workflow. When methods are called, the caller passes `session_type` and Memory routes to the correct session memory implementation.

### Initialization

```python
from upsonic.storage.memory import Memory
from upsonic.storage.redis import RedisStorage

storage = RedisStorage(db_url="redis://localhost:6379/0")

memory = Memory(
    storage=storage,
    session_id="session_001",
    user_id="user_123",
    full_session_memory=True,
    summary_memory=True,
    user_analysis_memory=True,
    model="openai/gpt-4o"
)
```

### Methods

#### prepare_inputs_for_task

Gather all relevant memory data before task execution.

```python
async def prepare_inputs_for_task(
    session_type: Optional[SessionType] = None,
    agent_metadata: Optional[Dict[str, Any]] = None,
) -> Dict[str, Any]
```

**Parameters:**
- `session_type` - The session type (defaults to AGENT)
- `agent_metadata` - Optional metadata from the caller to inject

**Returns:**
Dictionary with prepared memory inputs:
```python
{
    "message_history": [],  # List of messages
    "context_injection": "",  # Session summary
    "system_prompt_injection": "",  # User profile
    "metadata_injection": "",  # Session + agent metadata
}
```

**Example:**
```python
memory_inputs = await memory.prepare_inputs_for_task(
    session_type=SessionType.AGENT,
    agent_metadata={"agent_version": "1.0"}
)
```

#### save_session_async

Save session to storage. This is the centralized method for ALL session saving operations.

```python
async def save_session_async(
    output: AgentRunOutput,
    session_type: Optional[SessionType] = None,
    agent_id: Optional[str] = None,
) -> None
```

**Behavior:**
- For **INCOMPLETE** runs (paused, error, cancelled):
  - Saves checkpoint state for HITL resumption
  - Does NOT process memory features (summary, user profile)
- For **COMPLETED** runs:
  - Saves the completed run output
  - Generates session summary (if `summary_memory` enabled)
  - Analyzes user profile (if `user_analysis_memory` enabled)

**Parameters:**
- `output` - The run output (AgentRunOutput, TeamRunOutput, etc.)
- `session_type` - The session type (defaults to AGENT)
- `agent_id` - Optional agent identifier

**Example:**
```python
output = agent.do("Analyze this data")
await memory.save_session_async(output)
```

#### delete_session / delete_session_async

Delete the current or specified session.

```python

```

**Parameters:**
- `session_id` - Session ID to delete (defaults to current session)

#### load_resumable_run / load_resumable_run_async

Load a resumable run from storage by run_id.

```python
def load_resumable_run(
    run_id: str,
    session_type: Optional[SessionType] = None,
    agent_id: Optional[str] = None,
) -> Optional[RunData]

async def load_resumable_run_async(
    run_id: str,
    session_type: Optional[SessionType] = None,
    agent_id: Optional[str] = None,
) -> Optional[RunData]
```

Resumable runs include:
- **paused**: External tool execution pause
- **error**: Durable execution (error recovery)
- **cancelled**: Cancel run resumption

**Parameters:**
- `run_id` - The run ID to search for
- `session_type` - The session type (defaults to AGENT)
- `agent_id` - Optional agent_id to search across sessions

**Returns:** RunData if found and resumable, None otherwise

### Usage with Agent

```python
from upsonic import Agent
from upsonic.storage.memory import Memory
from upsonic.storage.postgres import PostgresStorage

# Create storage and memory
storage = PostgresStorage(db_url="postgresql://user:pass@localhost/db")
memory = Memory(
    storage=storage,
    full_session_memory=True,
    summary_memory=True,
    user_analysis_memory=True,
    model="openai/gpt-4o"
)

# Attach to agent
agent = Agent("openai/gpt-4o", memory=memory)

# Memory is used automatically during agent execution
result = agent.do("Analyze this document")
```

### Memory Features

#### Session Memory

- **Full Session Memory**: Persists complete chat history
- **Summary Memory**: Generates and stores conversation summaries
- **Metadata**: Stores custom metadata with sessions

#### User Memory

- **User Profile**: Extracts and maintains user preferences and traits
- **Dynamic Schema**: Automatically generates profile structure based on conversations
- **Update Modes**: Choose between merging updates or replacing entire profile

#### HITL Checkpointing

Memory automatically saves checkpoints for incomplete runs (paused, error, cancelled) to enable cross-process resumption, regardless of memory settings.
