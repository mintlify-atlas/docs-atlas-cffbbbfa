# Memory Management

Learn how to manage conversation history, session state, and user profiles using Upsonic's memory system. Enable agents to maintain context across interactions and remember important information.

## Memory System Overview

Upsonic's memory system provides:

- **Session Memory**: Conversation history and context
- **User Memory**: User profiles and preferences
- **Summary Generation**: Automatic context summarization
- **Multiple Storage Backends**: In-memory, file-based, database-backed
- **Cross-Process Persistence**: Resume conversations across restarts

Reference: `src/upsonic/storage/memory/memory.py:22-54`

## Memory Architecture

The `Memory` class orchestrates two types of memory:

1. **Session Memory**: Chat history, summaries, session metadata
2. **User Memory**: User profiles and traits extracted from conversations

```python
from upsonic.storage import InMemoryStorage
from upsonic.storage.memory import Memory

storage = InMemoryStorage()

memory = Memory(
    storage=storage,
    session_id="session_001",
    user_id="user_123",
    full_session_memory=True,
    summary_memory=True,
    user_analysis_memory=True
)
```

  <Step title="Configure Memory">
    Enable memory features:

    ```python
    memory = Memory(
        storage=storage,

        # Memory features
        full_session_memory=True,      # Store full chat history
        summary_memory=True,            # Auto-generate summaries
        user_analysis_memory=True,      # Extract user profile
        
        # Configuration

        debug=True
    )
    ```

  </Step>

  <Step title="Attach to Agent">

## Storage Backends

### In-Memory Storage

Fast but non-persistent:

```python
from upsonic.storage import InMemoryStorage

storage = InMemoryStorage()
memory = Memory(storage=storage)

# Data lost when process ends
```

**Use for**: Testing, development, disposable sessions

### JSON Storage

File-based persistence:

```python
from upsonic.storage import JSONStorage

storage = JSONStorage(
    base_path="./data/sessions"
)
memory = Memory(storage=storage)

# Data saved to JSON files
```

**Use for**: Small applications, simple persistence

### SQLite Storage

Local database:

```python
from upsonic.storage import SQLiteStorage

storage = SQLiteStorage(
    db_path="./data/sessions.db"
)
memory = Memory(storage=storage)
```

**Use for**: Desktop apps, local development

### Redis Storage

Distributed caching:

```python
from upsonic.storage import RedisStorage

storage = RedisStorage(
    host="localhost",
    port=6379,
    db=0,
    password="your_password"
)
memory = Memory(storage=storage)
```

**Use for**: Distributed systems, high-performance needs

### PostgreSQL Storage

Production database:

```python
from upsonic.storage import PostgreSQLStorage

storage = PostgreSQLStorage(
    connection_string="postgresql://user:password@localhost:5432/dbname"
)
memory = Memory(storage=storage)
```

**Use for**: Production applications, enterprise systems

### MongoDB Storage

Document database:

```python
from upsonic.storage import MongoDBStorage

storage = MongoDBStorage(
    connection_string="mongodb://localhost:27017",
    database_name="upsonic_db"
)
memory = Memory(storage=storage)
```

**Use for**: Flexible schema, large-scale applications

## Memory Features

### Full Session Memory

Store complete conversation history:

```python
memory = Memory(
    storage=storage,
    full_session_memory=True,  # Enable full history
    num_last_messages=50       # Keep last 50 messages
)

agent = Agent("openai/gpt-4o", memory=memory)

# All messages are remembered
agent.do(Task("What's the capital of France?"))
agent.do(Task("What did I just ask?"))  # Remembers previous question
```

Reference: `src/upsonic/storage/memory/memory.py:99`

### Summary Memory

Automatically generate conversation summaries:

```python
memory = Memory(
    storage=storage,
    summary_memory=True,
    model="openai/gpt-4o-mini"  # Used for summarization
)

agent = Agent("openai/gpt-4o", memory=memory)

# After many turns, summary is auto-generated
# and injected as context instead of full history
```

Reference: `src/upsonic/storage/memory/memory.py:100`

### User Profile Memory

Extract and remember user information:

```python
from pydantic import BaseModel

class UserProfile(BaseModel):
    name: str
    interests: list[str]
    preferences: dict

memory = Memory(
    storage=storage,
    user_analysis_memory=True,
    user_profile_schema=UserProfile,  # Structured profile
    user_memory_mode='update'  # 'update' or 'replace'
)

agent = Agent("openai/gpt-4o", memory=memory)

agent.do(Task("I love hiking and photography"))
# Profile updated with interests

agent.do(Task("Suggest activities for me"))
# Agent uses remembered interests
```

Reference: `src/upsonic/storage/memory/memory.py:102-110`

### Dynamic User Profiles

Let the system discover profile schema:

```python
memory = Memory(

    user_analysis_memory=True,
    dynamic_user_profile=True,  # Auto-discover schema
    model="openai/gpt-4o"
```

### Context Preparation

Memory prepares context for each task:

```python

# - Message history
# - Session summary (if enabled)
# - User profile (if enabled)
# - Session metadata

    session_type=SessionType.AGENT,
    agent_metadata={"version": "1.0"}
)

print(prepared["context_injection"])     # Summary
print(prepared["system_prompt_injection"]) # User profile
print(prepared["metadata_injection"])    # Metadata
```

### Cross-Process Resumption

Resume conversations across restarts:

```python
# Process 1: Start conversation
memory = Memory(
    storage=PostgreSQLStorage(conn_str),
    session_id="persistent_session"
)
agent = Agent("openai/gpt-4o", memory=memory)
agent.do(Task("Remember: my favorite color is blue"))

# Process 2: Resume later
memory = Memory(
    storage=PostgreSQLStorage(conn_str),
    session_id="persistent_session"  # Same session ID
)
agent = Agent("openai/gpt-4o", memory=memory)
agent.do(Task("What's my favorite color?"))  # "Blue"
```

## Best Practices

<Tip>
**Choose the Right Backend**: Match storage to your needs:

- **Development**: InMemoryStorage, SQLiteStorage
- **Production**: PostgreSQLStorage, RedisStorage
- **Scale**: MongoDBStorage for flexibility
</Tip>

<Tip>
**Limit Message History**: Prevent token overflow with message limits:

```python
memory = Memory(
    storage=storage,
    full_session_memory=True,
    num_last_messages=30  # Keep only recent 30 messages
)
```
</Tip>

<Warning>
**Privacy Concerns**: Be careful with user data:

- Use `user_analysis_memory` carefully in production
- Encrypt sensitive data at rest
- Implement data retention policies
- Comply with GDPR/privacy regulations
</Warning>

<Tip>
**Summary for Long Conversations**: Enable summary memory for efficiency:

```python
memory = Memory(
    storage=storage,
    summary_memory=True,  # Auto-summarize old messages
    model="openai/gpt-4o-mini"  # Cheaper model for summaries
)
```
</Tip>

<Note>
**Async Operations**: Memory supports async for better performance:

```python
# Async session operations
session = await memory.get_session_async()
messages = await memory.get_messages_async()
sessions = await memory.list_sessions_async()
```
</Note>

## Common Patterns

### Stateful Chatbot

```python
from upsonic import Agent, Task
from upsonic.storage import PostgreSQLStorage
from upsonic.storage.memory import Memory

storage = PostgreSQLStorage(connection_string=DB_URL)

def create_chat_agent(session_id: str, user_id: str):
    memory = Memory(
        storage=storage,
        session_id=session_id,
        user_id=user_id,
        full_session_memory=True,
        summary_memory=True,
        num_last_messages=20
    )
    
    return Agent(
        model="openai/gpt-4o",
        name="Assistant",
        memory=memory
    )

# Use in web app
user_id = request.user.id
session_id = request.session.id
agent = create_chat_agent(session_id, user_id)

response = agent.do(Task(user_message))
```

### Personalized Assistant

```python
from pydantic import BaseModel

```
