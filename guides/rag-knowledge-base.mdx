# RAG Knowledge Base

Learn how to build Retrieval-Augmented Generation (RAG) systems using Upsonic's KnowledgeBase. Enable your agents to access and retrieve information from documents, databases, and knowledge repositories.

## KnowledgeBase Overview

The `KnowledgeBase` class is the central orchestrator for RAG pipelines:

- **Intelligent Document Processing**: Automatic loader and splitter detection
- **Idempotent Operations**: Process documents once per configuration
- **Async-First Architecture**: High-performance async operations
- **Flexible Search**: Dense, full-text, and hybrid search
- **Tool Integration**: Use as agent tools automatically
- **Multiple Vector Databases**: Chroma, Qdrant, Milvus, PGVector, SuperMemory

Reference: `src/upsonic/knowledge_base/knowledge_base.py:27-45`

## Creating a Knowledge Base

<Steps>
  <Step title="Choose Vector Database">
    Select and configure a vector database:

    ```python
    from upsonic.rag.vectordb import (
        ChromaProvider,
        QdrantProvider,
        MilvusProvider,
        PGVectorProvider
    )
    from upsonic.rag.vectordb.config import (
        ChromaConfig,
        QdrantConfig
    )

    # Chroma (local development)
    chroma_config = ChromaConfig(
        collection_name="my_docs",
        persist_directory="./chroma_db"
    )
    vectordb = ChromaProvider(config=chroma_config)

    # Qdrant (production)
    qdrant_config = QdrantConfig(
        collection_name="my_docs",
        url="http://localhost:6333"
    )
    vectordb = QdrantProvider(config=qdrant_config)
    ```
  </Step>

  <Step title="Configure Embeddings">
    Choose an embedding provider:

    ```python
    from upsonic.rag.embeddings import (
        OpenAIEmbedding,
        CohereEmbedding,
        VoyageEmbedding
    )
    from upsonic.rag.embeddings.config import OpenAIEmbeddingConfig

    # OpenAI embeddings
    embedding_config = OpenAIEmbeddingConfig(
        model="text-embedding-3-large",
        api_key="your_openai_key"
    )
    embeddings = OpenAIEmbedding(config=embedding_config)
    ```
  </Step>

  <Step title="Create Knowledge Base">
    Initialize with sources:

    ```python
    from upsonic.knowledge_base import KnowledgeBase

    kb = KnowledgeBase(
        sources=["./docs/", "./README.md"],
        vectordb=vectordb,
        embedding_provider=embeddings,
        name="technical_docs",
        description="Technical documentation for the project"
    )

    # Setup (idempotent - safe to call multiple times)
    await kb.setup_async()
    ```

    Reference: `src/upsonic/knowledge_base/knowledge_base.py:47-156`
  </Step>

  <Step title="Query Knowledge Base">
    Retrieve relevant information:

    ```python
    # Search for information
    results = await kb.query_async(
        query="How do I configure authentication?",
        top_k=5
    )

    for result in results:
        print(f"Score: {result.score}")
        print(f"Text: {result.text}")
        print(f"Source: {result.metadata['source']}")
    ```
  </Step>
</Steps>

### Chunking Strategies

```python
from upsonic.rag.text_splitter import (
    RecursiveChunker,
    CharacterChunker,
    SemanticChunker,
    PythonChunker,
    MarkdownHeaderChunker,
    HTMLChunker,
    JSONChunker,
    AgenticChunker
)

# Recursive chunking (default)
recursive = RecursiveChunker(
    chunk_size=1000,
    chunk_overlap=200
)

# Semantic chunking (content-aware)
semantic = SemanticChunker(
    embedding_provider=embeddings,
    buffer_size=1
)

# Python code chunking
python = PythonChunker(
    chunk_size=1500,
    chunk_overlap=200
)

kb = KnowledgeBase(
    sources=["./code/"],
    splitters=[python],  # Custom chunker
    vectordb=vectordb,
    embedding_provider=embeddings
)
```

## Using with Agents

### As a Tool

KnowledgeBase automatically becomes a searchable tool:

```python
from upsonic import Agent, Task
from upsonic.knowledge_base import KnowledgeBase

# Create knowledge base
kb = KnowledgeBase(
    sources=["./docs/"],
    vectordb=vectordb,
    embedding_provider=embeddings,
    name="product_docs",
    description="Product documentation and guides"
)
await kb.setup_async()

# Agent can use it as a tool
agent = Agent(
    model="openai/gpt-4o",
    tools=[kb]  # KnowledgeBase acts as a tool
)

# Agent automatically searches when needed
result = agent.do(
    Task("How do I configure the database?")
)
```

Reference: `src/upsonic/knowledge_base/knowledge_base.py:149-255`

### Dynamic Tool Names

Each KnowledgeBase gets a unique tool name:

```python
# Multiple knowledge bases
technical_kb = KnowledgeBase(
    name="technical_docs",
    sources=["./technical/"],
    vectordb=vectordb1,
    embedding_provider=embeddings
)

user_kb = KnowledgeBase(
    name="user_guides",
    sources=["./guides/"],
    vectordb=vectordb2,
    embedding_provider=embeddings
)

# Agent gets both as tools:
# - search_technical_docs(query: str)
# - search_user_guides(query: str)
agent = Agent(
    model="openai/gpt-4o",
    tools=[technical_kb, user_kb]
)
```

Reference: `src/upsonic/knowledge_base/knowledge_base.py:180-255`

### Task-Level Configuration

Configure search per task:


## Best Practices

<Tip>
**Chunk Size**: Choose appropriate chunk sizes:

- **Small chunks (200-500 tokens)**: Better precision, more chunks
- **Medium chunks (500-1000 tokens)**: Balanced
- **Large chunks (1000-2000 tokens)**: More context, fewer chunks

```python
kb = KnowledgeBase(
    sources=["./docs/"],
    vectordb=vectordb,
    embedding_provider=embeddings,
    splitter_config={
        "chunk_size": 800,  # Balanced size
        "chunk_overlap": 150  # 15-20% overlap
    }
)
```
</Tip>

<Tip>
**Quality Preference**: Set based on your needs:

```python
# Fast processing, good enough quality
kb = KnowledgeBase(
    sources=["./docs/"],
    vectordb=vectordb,
    embedding_provider=embeddings,
    quality_preference="fast"
)

# Best quality, slower processing
kb = KnowledgeBase(
    sources=["./docs/"],
    vectordb=vectordb,
    embedding_provider=embeddings,
    quality_preference="quality"
)
```
</Tip>

<Warning>
**Embedding Costs**: Be mindful of embedding API costs:

- Use smaller, cheaper embedding models for development
- Cache embeddings when possible
- Monitor usage in production

```python
# Development: cheaper model
dev_embeddings = OpenAIEmbedding(
    config=OpenAIEmbeddingConfig(
        model="text-embedding-3-small"
    )
)

# Production: better quality
prod_embeddings = OpenAIEmbedding(
    config=OpenAIEmbeddingConfig(
        model="text-embedding-3-large"
    )
)
```
</Warning>

<Tip>
**Metadata Strategy**: Add useful metadata for filtering:

```python
# Loaders can add custom metadata
loader = PDFLoader()
loader.metadata_extractor = lambda doc: {
    "category": extract_category(doc),
    "version": extract_version(doc),
    "author": extract_author(doc)
}
```
</Tip>

<Note>
**Async Operations**: Always use async for better performance:

```python
# Good: Async
await kb.setup_async()
results = await kb.query_async(query)

# Acceptable: Sync wrapper (uses async internally)
results = kb.query(query)  # Calls query_async internally
```
</Note>

## Common Patterns

### Documentation Assistant

```python
# Index documentation
kb = KnowledgeBase(
    sources=["./docs/", "./README.md"],
    vectordb=vectordb,
    embedding_provider=embeddings,
    name="docs",
    description="Product documentation",
    topics=["Getting Started", "API Reference", "Tutorials"]
)
await kb.setup_async()

# Create assistant
assistant = Agent(
    model="openai/gpt-4o",
    name="Documentation Assistant",
    role="Help users understand the product",
    tools=[kb],
    system_prompt="Use the documentation to answer questions accurately."
)
```

### Code Search

```python
from upsonic.rag.text_splitter import PythonChunker

# Index Python codebase
kb = KnowledgeBase(
    sources=["./src/"],
    vectordb=vectordb,
    embedding_provider=embeddings,
    splitters=[PythonChunker(chunk_size=1500)],
    name="codebase",
    description="Python source code"
)
await kb.setup_async()

# Code search agent
code_agent = Agent(
    model="openai/gpt-4o",
    tools=[kb],
    system_prompt="Help developers find code examples and understand the codebase."
)
```

### Multi-Source RAG

```python
# Multiple knowledge bases
internal_kb = KnowledgeBase(
    name="internal_docs",
    sources=["./internal/"],
    vectordb=vectordb1,
    embedding_provider=embeddings
)

public_kb = KnowledgeBase(
    name="public_docs",
    sources=["./public/"],
    vectordb=vectordb2,
    embedding_provider=embeddings
)

# Agent with multiple sources
agent = Agent(
    model="openai/gpt-4o",
    tools=[internal_kb, public_kb],
    system_prompt="Search both internal and public documentation."
)
```
