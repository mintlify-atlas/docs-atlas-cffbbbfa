## Overview

`AutonomousAgent` is a specialized agent that inherits from `Agent` and comes pre-configured with filesystem and shell capabilities. It's the ideal choice for coding assistants, DevOps automation, system administration, and any task requiring file or shell access.

## Key Features

- **Default Storage & Memory**: Automatically creates InMemoryStorage and Memory if not provided
- **Filesystem Tools**: Read, write, edit, search, list, move, copy, delete files
- **Shell Tools**: Execute terminal commands with timeout and output capture
- **Workspace Sandboxing**: All operations are restricted to a designated workspace directory
- **Security-First**: Built-in command filtering and path validation
- **Greeting Workflow**: Optional automatic greeting when workspace is set

## Basic Usage

### Simple Autonomous Agent

```python
from upsonic import AutonomousAgent

# Create agent with workspace
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/path/to/project"
)

# Execute file operations
result = agent.do("Read the main.py file and add error handling")
```

### Agent with Full Session Memory

```python
agent = AutonomousAgent(
    model="anthropic/claude-sonnet-4-20250514",
    workspace="/path/to/project",
    name="CodeAssistant",
    full_session_memory=True,  # Enable chat history
    summary_memory=True  # Enable session summaries
)

# Multi-turn coding assistance
agent.do("List all Python files in the project")
agent.do("Read the main.py file")
agent.do("Add logging to the main function")
```

## Built-in Tools

### Filesystem Tools

The agent automatically includes these filesystem tools:

- **`read_file(path, offset, limit)`**: Read file contents with pagination support
- **`write_file(path, content)`**: Create or overwrite files (creates parent directories)
- **`edit_file(path, old_string, new_string)`**: Make precise text replacements
- **`list_files(path, pattern)`**: List directory contents with filtering
- **`search_files(pattern)`**: Find files by name pattern
- **`grep_files(pattern, path)`**: Search for text/regex in file contents
- **`file_info(path)`**: Get file metadata (size, permissions, modification time)
- **`create_directory(path)`**: Create directories
- **`move_file(source, dest)`**: Move or rename files/directories
- **`copy_file(source, dest)`**: Copy files or directories
- **`delete_file(path)`**: Delete files or directories

### Shell Tools

The agent includes these shell execution tools:

- **`run_command(command, timeout)`**: Execute shell commands
- **`run_python(code)`**: Execute Python code snippets
- **`check_command_exists(command)`**: Verify command availability

## Security Features

### Workspace Sandboxing

All file and shell operations are restricted to the workspace:

```python
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/safe/project"
)

# This works - inside workspace
agent.do("Read /safe/project/data.txt")

# This is blocked - path traversal attempt
agent.do("Read /etc/passwd")  # Blocked by path validation
```

### Command Filtering

Dangerous commands are blocked by default:

```python
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/path/to/project",
    blocked_commands=["rm -rf /", "dd", "mkfs"]
)
```

### File Access Tracking

The filesystem toolkit tracks which files have been read:

```python
# Reset tracking if needed
agent.reset_filesystem_tracking()
```

## Advanced Features

### Custom System Prompt

The agent builds a dynamic system prompt based on enabled tools. You can also provide a custom one:

```python
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/path/to/project",
    system_prompt="You are a Python expert. Focus on writing clean, maintainable code."
)
```

### Workspace Greeting

When workspace is set, the agent can generate a greeting message:

```python
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/path/to/project"  # Loads AGENTS.md if present
)

# Execute greeting (done automatically before first task)
greeting = agent.execute_workspace_greeting()
print(greeting)  # "Hello! What would you like to work on today?"
```

### Heartbeat Execution

For long-running sessions or interfaces:

```python
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/path/to/project",
    heartbeat=True,
    heartbeat_period=30,  # minutes
    heartbeat_message="Provide a brief status update"
)

# Execute heartbeat manually
status = agent.execute_heartbeat()
```

### Storage and Memory Properties

Access the auto-created storage and memory:

```python
agent = AutonomousAgent(
    model="openai/gpt-4o",
    workspace="/path/to/project"
)

# Access storage
print(agent.autonomous_storage)  # InMemoryStorage instance

# Access memory
print(agent.autonomous_memory)  # Memory instance

# Access toolkits
print(agent.filesystem_toolkit)  # AutonomousFilesystemToolKit
print(agent.shell_toolkit)  # AutonomousShellToolKit
```

## Complete Example

```python
from upsonic import AutonomousAgent

# Create autonomous coding assistant
agent = AutonomousAgent(
    model="anthropic/claude-sonnet-4-20250514",
    workspace="/home/user/my-project",
    name="CodeAssistant",
    role="Senior Python Developer",
    goal="Help maintain and improve the codebase",
    
    # Memory
    full_session_memory=True,
    summary_memory=True,
    
    # Tools
    enable_filesystem=True,
    enable_shell=True,
    shell_timeout=60,
    
    # Security
    blocked_commands=["rm -rf", "dd"],
    
    # Debug
    debug=True
)

# Multi-step task
task = """
1. Read the main.py file
2. Add error handling to all functions
3. Run the test suite
4. If tests pass, commit the changes
"""

result = agent.do(task)
print(result)
```

## System Prompt Structure

The agent automatically generates a comprehensive system prompt:

```text
<AutonomousAgent>
You are an Autonomous Agent with access to tools for completing tasks within your designated workspace.

## Your Capabilities

### Filesystem Tools
You have comprehensive filesystem access within the workspace:
- read_file: Read file contents
- write_file: Create new files or overwrite
- edit_file: Make precise text replacements
...

### Shell Tools
You can execute commands in the workspace directory:
- run_command: Execute shell commands
- run_python: Execute Python code snippets
...

## Guidelines

### When to Use Filesystem Tools
- Reading code: Use read_file before making changes
- Creating files: Use write_file for new files
...

## Best Practices
1. Read before edit
2. Use precise edits
3. Check command availability
4. Handle errors gracefully
5. Work incrementally

## Security Restrictions
- All file operations are sandboxed to the workspace
- Path traversal outside workspace is blocked
- Dangerous shell commands are blocked
</AutonomousAgent>
```
