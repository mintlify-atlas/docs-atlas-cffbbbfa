## Overview

Custom tools allow you to extend your agents with any functionality you need. Upsonic provides several ways to create tools, from simple function decorators to full toolkit classes.

## Function tools with @tool

The simplest way to create a tool:

```python
from upsonic import Agent, Task, tool

@tool
def calculate_revenue(sales: float, costs: float) -> float:
    """Calculate net revenue from sales and costs."""
    return sales - costs

agent = Agent(model="anthropic/claude-sonnet-4-5")
task = Task(
    description="Calculate revenue with $10000 sales and $6000 costs",
    tools=[calculate_revenue]
)

result = agent.do(task)
print(result)  # "The net revenue is $4000"
```

<Tip>
The docstring becomes the tool description that the LLM sees. Write clear, descriptive docstrings.
</Tip>

## ToolKit classes

Group related tools into a toolkit:

```python
from upsonic.tools import ToolKit
import requests

class WeatherToolKit(ToolKit):
    def __init__(self, api_key: str):
        super().__init__()
        self.api_key = api_key
        self.base_url = "https://api.weather.com"
    
    @tool
    def get_current_weather(self, city: str) -> dict:
        """Get current weather for a city."""
        response = requests.get(
            f"{self.base_url}/current",
            params={"city": city, "key": self.api_key}
        )
        return response.json()
    
    @tool
    def get_forecast(self, city: str, days: int = 7) -> dict:
        """Get weather forecast for a city."""
        response = requests.get(
            f"{self.base_url}/forecast",
            params={"city": city, "days": days, "key": self.api_key}
        )
        return response.json()

# Use the toolkit
weather_tools = WeatherToolKit(api_key="your-key")

task = Task(
    description="What's the weather in San Francisco?",
    tools=[weather_tools]
)
```

## Tool with configuration

Configure tool behavior:

```python
from upsonic import tool
from upsonic.tools import ToolConfig

@tool(config=ToolConfig(
    cache_enabled=True,
    timeout=30,
    retry_count=3
))
def fetch_data(url: str) -> str:
    """Fetch data from a URL."""
    import requests
    response = requests.get(url, timeout=30)
    return response.text
```

## Async tools

Create async tools for I/O operations:

```python
import aiohttp
from upsonic import tool

@tool
async def fetch_async(url: str) -> str:
    """Asynchronously fetch data from a URL."""
    async with aiohttp.ClientSession() as session:
        async with session.get(url) as response:
            return await response.text()

# Use with async agent
task = Task(
    description="Fetch data from https://api.example.com",
    tools=[fetch_async]
)

result = await agent.do_async(task)
```

## Tool with type validation

Use Pydantic for input validation:

```python
from pydantic import BaseModel, Field
from upsonic import tool

class SearchParams(BaseModel):
    query: str = Field(..., description="Search query")
    limit: int = Field(10, ge=1, le=100, description="Number of results")
    include_snippets: bool = Field(True, description="Include text snippets")

@tool
def search_documents(params: SearchParams) -> list:
    """Search documents with validated parameters."""
    # params.query, params.limit, params.include_snippets are validated
    results = perform_search(params.query, params.limit)
    return results
```

## Error handling in tools

Handle errors gracefully:

```python
from upsonic import tool

@tool
def divide_numbers(a: float, b: float) -> float:
    """Divide two numbers."""
    if b == 0:
        raise ValueError("Cannot divide by zero")
    return a / b
```

The agent will see the error message and can adjust its approach.

## Database access tool

Create a tool for database operations:

```python
import sqlite3
from upsonic.tools import ToolKit, tool

class DatabaseToolKit(ToolKit):
    def __init__(self, db_path: str):
        super().__init__()
        self.db_path = db_path
    
    @tool
    def query_users(self, email: str = None) -> list:
        """Query users from database."""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        
        if email:
            cursor.execute("SELECT * FROM users WHERE email = ?", (email,))
        else:
            cursor.execute("SELECT * FROM users LIMIT 10")
        
        results = cursor.fetchall()
        conn.close()
        return results
    
    @tool
    def count_orders(self, user_id: int) -> int:
        """Count orders for a user."""
        conn = sqlite3.connect(self.db_path)
        cursor = conn.cursor()
        cursor.execute("SELECT COUNT(*) FROM orders WHERE user_id = ?", (user_id,))
        count = cursor.fetchone()[0]
        conn.close()
        return count

db_tools = DatabaseToolKit("./app.db")
task = Task(
    description="How many orders does user 123 have?",
    tools=[db_tools]
)
```

## Best practices

<Warning>
**Security**: Validate inputs and sanitize outputs. Never execute arbitrary code from user input.
</Warning>

<Tip>
**Error messages**: Return clear error messages that help the agent understand what went wrong.
</Tip>

<Note>
**Performance**: Keep tools fast. Use caching for expensive operations.
</Note>

See the [Tool API reference](/api/tools/base) for complete documentation.
